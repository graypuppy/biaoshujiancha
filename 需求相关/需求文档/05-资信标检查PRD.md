# **模块概述**
## **核心定位**
资格审查，自动检查投标文件资信部分的合规性，降低因资格问题导致的废标风险。
## **解决的核心问题**

|   |   |   |
|---|---|---|
|问题|现状|本模块的解决方案|
|**人工检查效率低**|一份标书人工检查需2-3小时|AI自动检查，5分钟内完成|
|**易疏漏**|项目多、材料更新频繁，人工易遗漏|系统自动比对所有检查点，无遗漏|
|**废标损失大**|一个小错误（如证书过期3天）导致废标|提前发现所有问题，避免废标|
|**专业能力不足**|小微企业不清楚检查哪些内容|系统自动提取招标要求，引导检查|

## **确认检查文件**
### 单文件
第一步预览被检查文件

**页面**
- 仅展示本次将被检查的投标文件预览页（PDF阅读器）。
- 页面底部固定操作区：**确认并进入下一步** / **返回**。

**内容**
- 文件信息：文件名
- 提示文案：系统已自动选中唯一版本，请确认文件是否为本次要检查的投标文件。

**交互**
- 存在多份时可切换
- 预览能力：翻页、缩放、目录、搜索。
- 操作按钮：
  - **确认并进入下一步**：进入“确认评分规则/检查点”页面。
  - **返回**：回到首页（退出前弹窗确认）。

**业务规则**
- 单文件场景默认自动选中并跳过“多版本选择”，但仍需给用户一次预览确认机会。
- 未点击“确认并进入下一步”不进入后续流程（避免误检错误文件）。
- 若文件加载失败/权限不足/解析失败，需阻断流程并提示用户重新上传或更换文件。

### 多文件
第一步在弹窗中确认要检查的文件，默认全选

**弹窗**
- 展示文件列表，如图所示，取消勾选则不检查该文件
- 页面底部固定操作区：**确认并进入下一步** / **返回**

**内容**
- 文件列表项包含：
  - 复选框（默认全选）
  - 文件名（如："投标文件_V1.0.pdf"）
  - 上传时间（如："2026-02-05 10:30"）
  - 文件大小（如："12.5MB"）
- 提示文案："系统检测到 {N} 个投标文件版本可检查，请选择要检查的文件"
- 已选文件数量提示："已选择 {N} 个文件"

**交互**
- 点击文件列表项：自动勾选/取消勾选该文件
- 复选框操作：勾选/取消勾选文件，至少保留1个选中
- 全选/取消全选：列表顶部提供快捷操作按钮
- 操作按钮：
  - **确认并进入下一步**：对已选文件进入"确认评分规则/检查点"页面
  - **返回**：回到首页（退出前弹窗确认："检查尚未开始，确认返回吗？"）

**业务规则**
- 默认全选所有版本，支持用户取消部分版本（批量检查场景）
- 至少选择1个文件才能进入下一步，否则"确认并进入下一步"按钮置灰不可点击
- 若有文件加载失败/解析失败，该文件显示异常状态图标，但不阻断其他文件的选择和检查
- 批量检查时，后续检查结果页面需支持按文件版本分组展示结果

## **检查过程**

**业务目标**: 将投标文件的信息与招标要求进行比对，判断是否符合要求
### 前端用户操作

**页面**
- 全屏检查进度页面
- 进度条显示检查进度（0-100%）
- 当前检查项提示（如："正在检查营业执照有效期..."）
- 预计剩余时间提示

**内容**
- 检查进度百分比
- 当前正在检查的项目名称
- 已完成/总数（如："5/10"）

**交互**
- 无用户操作，系统自动执行检查
- 检查完成后自动跳转到检查结果页面

**业务规则**
- 检查过程不可中断（避免数据不完整）
- 预计时间根据检查点数量动态计算
- 检查失败时显示错误提示，允许重新检查


### **后端逻辑**

后端检查流程分为三个步骤：

#### 1. 解析投标文件

**业务目标**: 从投标文件中提取企业名称、证照信息、业绩、人员等关键信息
**基本流程**:

```Plain
用户点击"开始检查"
    ↓
系统解析投标文件
    ↓
OCR识别证照图片（营业执照、资质证书等）
    ↓
大模型提取关键信息
    ↓
结构化存储（用于后续比对）
    ↓
进入检查阶段
```

**提取的关键信息**:
- 企业基本信息（名称、统一社会信用代码） 
- 营业执照（企业名称、有效期）
- 资质证书（证书名称、等级、有效期）
- 安全许可证（许可证号、有效期）
- 业绩证明（项目名称、合同金额、日期）  
- 人员信息（姓名、证书类型、等级、有效期、社保）  
- 投标保证金（金额、形式）  
- 投标报价（总价） 
- 工期承诺、质量标准、盖章签字情况  

**关键点**:
1. **OCR** **+ AI结合**: 证照通常是图片，需要先OCR识别，再用AI理解
2. **位置记录**: 记录每个信息的来源页码，便于定位问题
3. **容错机制**: 部分信息提取失败不阻塞整体检查，标记为"未提取到"

**注意事项**:
- 图片模糊可能导致OCR失败，需要提示用户提供清晰图片
- 非标准格式的证书可能识别率低，需要人工复核
    

#### 2. 比对与检查

**检查逻辑**:

自动检查项:

```Plain
加载评审规则
    ↓
加载投标文件提取的信息
    ↓
逐项比对
    ↓
简单规则（如: 日期比较） → 调用算子包
复杂语义（如: 资质类型判断） → 调用大模型
    ↓
输出检查结果（通过/不通过 + 原因 + 位置）
```

人工检查项:

```Plain
系统无法自动判断的检查项
    ↓
在检查结果中标记为"待人工填写"
    ↓
用户查看投标文件后，手动填写结果（通过/不通过）
    ↓
用户填写原因（可选）
```

**检查结果状态**:

- ✓ 通过: 符合要求
- ✗ 不通过: 不符合要求（致命错误）
- ⚠ 警告: 有问题但不致命
- ○ 待人工填写: 人工检查项，等待用户填写
- ⊘ 跳过: 无法检查（如: 投标文件未提供该信息）
    

**关键设计点**:

1. **问题定位**: 所有问题都标注来源页码和章节
2. **原因说明**: 用对比的方式说明（要求 vs 实际）
3. **严重程度**: 区分致命错误（导致废标）和一般警告
4. **修复建议**: 提供具体的、可操作的修复建议
    

**检查示例**:

**示例1: 营业执照有效期检查**

```Plain
要求: 有效期晚于投标截止日期(2026-02-10)
实际: 营业执照有效期 2027-05-20
结果: ✓ 通过
原因: 营业执照有效期(2027-05-20)晚于投标截止日期(2026-02-10)
位置: 第5页 "第一章 资格证明文件"
```

**示例2: 资质等级检查**

```Plain
要求: 市政公用工程施工总承包 ≥ 贰级
实际: 建筑工程施工总承包 贰级
结果: ✗ 不通过（致命错误）
原因: 资质类型不匹配。要求"市政公用工程施工总承包"，实际提供"建筑工程施工总承包"
建议: 请提供符合要求的市政公用工程施工总承包资质证书
位置: 第7页 "第一章 资格证明文件"
```

**示例3: 人工检查项**

```Plain
检查点: 其他资料完整性
要求: 提供各省份要求的其他资料
结果: ○ 待人工填写
说明: 此项需要您对照招标文件人工核对材料是否齐全
操作: 请在检查结果页面填写检查结果
```

#### 3. 生成检查结果

**输出内容**:
- 检查结果列表（每个检查点的状态、原因、位置）
- 统计信息（通过/不通过/警告/待填写的数量）
- 问题汇总（按严重程度排序）

**数据结构**:
- 检查点ID、名称、状态、原因、来源页码、是否否决项
- 支持按文件版本分组（多文件检查场景）

**返回前端**:
- 实时推送检查进度（WebSocket或轮询）
- 检查完成后返回完整结果数据
- 自动跳转到检查结果展示页面

  - 问题类型标识（红色叉号图标）
  - 问题描述（如："存在不符合项的"）
  - 具体问题数量（如："23个检查点不符合、2个检查点符合、2个检查点是人工检查项"）
  - 警示文案（如："检查结果仅供参考，请以具体检查结果为准。人工检查项，请自己填写！"）
- **检查结果列表**（表格列）：
  - 序号：检查项编号（支持展开/收起子项）

**交互**
- 点击序号前的展开/收起图标 → 展开/收起该检查项的子项

**业务规则**
- 默认展示所有检查项，优先展示不通过的项（按严重程度排序）
- 存在子项的检查点默认收起，用户可手动展开查看详情
- 状态标识规则：
  - ✓ 绿色对勾 = 通过
  - ✗ 红色叉号 = 不通过（致命错误）
  - ⚠ 橙色警告 = 警告（非致命）
  - ○ 蓝色圆圈 = 待人工填写
  - ⊘ 灰色斜杠 = 跳过（无法检查）
- 人工检查项需用户手动填写结果后才能完成整检查
- **顶部状态卡片展示规则**：

系统首先统计各类检查结果的数量，然后根据优先级判断文件的整体检查状态，最后展示对应的状态卡片。

**第一步：统计检查结果**

系统需要统计以下5种检查结果的数量：
- `failCount` = 不通过（✗）的检查项数量
- `warningCount` = 警告（⚠）的检查项数量
- `passCount` = 通过（✓）的检查项数量
- `pendingCount` = 待人工填写（○）的检查项数量
- `skipCount` = 跳过（⊘）的检查项数量

**第二步：判断文件检查状态**

根据统计结果，按优先级从高到低判断文件的整体检查状态，并展示对应的状态卡片：

|优先级|状态|判断条件|提示文案|统计文案|警示文案|
|---|---|---|---|---|---|
|1|存在不符合项|`failCount > 0`|存在不符合项的|{failCount}个检查点不符合、{passCount}个检查点符合、{pendingCount}个检查点是人工检查项|检查结果仅供参考，请以具体检查结果为准。人工检查项，请自己填写！|
|2|存在警告项|`failCount == 0 && warningCount > 0`|存在需要注意的项|{warningCount}个检查点需注意、{passCount}个检查点符合、{pendingCount}个检查点是人工检查项|以下项目建议优化，但不影响投标资格|
|3|存在人工检查项|`failCount == 0 && warningCount == 0 && pendingCount > 0`|待您完成人工检查|{pendingCount}个检查点待人工填写、{passCount}个检查点已通过|请对照投标文件完成人工检查项，填写后可查看完整检查结果|
|4|全部正常|`failCount == 0 && warningCount == 0 && pendingCount == 0 && passCount > 0`|检查全部通过|{passCount}个检查点全部符合要求|恭喜！所有检查项均符合招标文件要求|

**第三步：补充信息**

- **跳过项提示**：如果存在跳过项（`skipCount > 0`），在任何状态卡片底部追加提示："有{skipCount}个检查点因投标文件未提供相关信息而跳过"
- **多文件场景**：如果是多文件检查场景，在卡片顶部显示文件选择器："当前查看：投标文件_V1.0.pdf"（下拉可切换其他文件）

#### 1. 检查结果持久化存储

**数据存储**：

- 存储内容：

**查询接口**：
- 根据项目ID和文件ID查询检查结果
- 支持按状态筛选（仅不通过/仅警告/仅待填写等）
- 支持分页查询（列表视图）

#### 2. 文件视图的页面检查项映射

**功能**：根据文件页码快速查询该页的检查项

**实现方式**：
- 在生成检查结果时，为每个检查项建立页码索引
- 提供查询接口：`GET /api/check-results/{fileId}/page/{pageNumber}`
- 返回该页面相关的所有检查项

**数据结构示例**：
```json
{
  "pageNumber": 5,
  "checkItems": [
    {
      "id": "CP-001",
      "name": "营业执照有效期",
      "status": "pass",
      "result": "营业执照有效期(2027-05-20)晚于投标截止日期(2026-02-10)",
      "location": "第5页 第一章 资格证明文件"
    }
  ]
}
```

**导出内容**：
- 项目基本信息
