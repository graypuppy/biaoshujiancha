1. 产品背景
详见完整产品市场洞察报告：标书检查产品市场洞察报告
1.1 现状与洞察背景
标书检查产品作为投标人服务中的主要增值内容，带来了持续和稳定的收益。目前我司产品功能聚焦“本单位”、“经济标”、“技术标暗标格式检查”等，2022-2025年销售毛利稳步上升，2025年预估收益已超500万元。但目前产品形态多样，功能过于分散，营销和培训成本高，用户难以获得完整的产品体验。
1.2 行业趋势与市场空间
- 市场体量巨大：全国公共资源交易体量基本保持在145w宗左右。其中工程建设板块累计约30w宗，按照我司市场份额及付费意愿测算，整体市场空间约1.8亿/年，我司机会空间预计在1亿/年左右。
- 监管政策加码：打击围标、串标行为已成为监管核心。投标文件机器特征码、文件属性、内容雷同、错误相似度等维度的检查手段逐渐丰富，对投标人的技术门槛要求越来越高。
- 技术可行性：大语言模型在合同审查、论文评审等领域的成熟应用，证明了其在风险定位、条款建议、语义分析方面的深度理解能力。将此技术迁移至“投标方AI自查”已具备明确的技术基础和实践路径。
1.3 客户痛点分析
基于对2000个废标案例的深度分析，资质标废标占比最高（77%以上），其次是经济标和技术标。
- 中大企业：项目多、材料更新频繁，人工检查成本高且易疏漏，一旦废标损失巨大。
- 小微企业：专业能力不足，极易触发围串标监测，且对价格敏感，需要高性价比的合规兜底服务。
1.4 竞品差距与挑战
目前友商（如广联达云清标、知也云、链企AI）已形成相对完整的解决方案。我司在商务标AI检查、经济标横向比对、技术标文件属性检查、模拟算分、标书深度查重、私有化部署层面存在较大差距。   
1.5 演进目标
本产品拟整合已有产品和检测能力点，形成完整的标书检查解决方案。通过提升产品力（新增资信标检查、模拟算分、技术标响应性检查等）及优化分级定价模式，实现应用端与能力端剥离，构建具备私有化部署能力的标书智检生态。

- 产品层面应考虑功能和定价分级：基础服务服务所有用户，高级风控（围串标检查，模拟算分）面向特定用户，深度检查（复杂技术标评估）面向付费意愿强的大中企业。
- 服务模式差异化：向小微企业提供SaaS化轻量服务；向大中企业提供私有化部署或深度集成方案。
3. 战略定位与核心愿景
- 从“单点工具”转向“风险管控中台”：打破原有产品分散现状，构建涵盖资信、经济、技术全维度的“一站式”标书检查。
- 以“项目”为核心的工作流引擎：以项目驱动，确保从招标文件解析到最终风险评估的全流程业务闭环。

需处理大文本接收，考虑从客户端只传提取后的文本而非整个大文件（除非做存档）。
AI 大模型

智能体 (Agent)
1. 语义分析：处理非结构化文本、条款理解。
2. 智能调用：通过 Tool Calling 调用能力平台的算子包。
3. 知识检索：内置 RAG 引擎检索法规与规范。
专注于“思考”与“编排”，将计算外包给算子。
能力平台 (架构部)
基础设施 & 算子提供方
1. 算子包：提供经过验证的计算/逻辑算子（如公式计算、强规则校验）。
2. 基础能力：提供统一支付、统一用户等通用能力供业务方接入。
3. 直连服务：支持业务端直接调用算子进行确定性检查。
算子包需具备原子性，既支持被模型调用，也支持被 API 直接调用。

5.2 详细功能模块设计
5.2.1 客户端核心模块
5.2.1.1 登陆与鉴权模块
- 用户打开客户端，输入账号密码。
- 客户端请求标桥接口验证身份，并获取当前用户的会员权益（是否会员、剩余次数）。
- 按次购买控制：
  - 在点击“开始检查”时触发。
  - 客户端需处理“余额不足”的逻辑，调用支付接口或弹窗提示。
5.2.1.2 文件解析与地区识别
- 地区识别：启动时调用标桥接口，获取支持的地区列表。用户选择或系统自动匹配标书地区。
- 文件解析：支持导入招投标文件或工程量清单文件。
  - 待确认：是否需要对标书 PDF 提取文字层，结构化数据等操作。
5.2.1.3 清标
此模块不依赖 AI，运行在本地，调用清标组件进行数学和逻辑校验： 
1. 清标内容：清单符合性检查、计算错误检查、取费检查等。
2. 输出：清标数据、结果数据库。
5.2.2 服务端与 AI 交互模块
5.2.2.1 数据接收与预处理
- 接口层接收来自客户端的数据：
  1. 创建任务：接收任务申请，返回 TaskID。
  2. 待检查文本：接收客户端分块上传的结构化文本。
  3. 注：本地清标结果可选择性异步上传存档，但不阻塞展示流程。
5.2.2.2 能力平台 / AI 大模型检查逻辑
系统采用AI 大模型 + 能力平台的双核驱动模式：
1. 能力平台 (架构部)： 作为基础设施，提供确定性的计算与检索能力。
  - 基础服务能力：
统一用户：提供统一的用户身份认证等能力。
统一支付：提供标准的支付接口等能力。
- 算子包：
内容：封装通用的数学公式、逻辑校验、数据提取规则。
双重服务：
2. 被 AI 调用：作为 Tool 供大模型在推理过程中调用（如验证文本中的金额计算是否正确）。
3. 被 API 调用：对于无需语义理解的规则检查，服务端 API 直接调用算子包，效率更高。
4. AI 大模型服务： 作为智能体 (Agent)，负责处理复杂逻辑。
  - 语义理解：分析招标文件中的非结构化条款。
  - 知识增强 (RAG)：内置向量检索模块，可检索相关法律法规、行业规范补充上下文。
  - Tool Calling：在分析过程中，动态识别需要计算或查证的环节，自动调用能力平台的算子包，并将结果整合输出。
价值：通过调用外部算子，解决大模型”一本正经胡说八道”的幻觉问题。
