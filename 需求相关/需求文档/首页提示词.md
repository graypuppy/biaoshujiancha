# 标书检查系统首页设计提示词

## 📋 提示词使用说明

本提示词可用于生成标书检查系统首页的前端代码(React/HTML)或设计稿。建议分模块生成,逐步完善。

---

## 🎯 核心提示词(完整页面)

```
请设计并实现一个企业级标书检查系统的首页,面向标书制作人员使用。页面需要专业、高效、信息密度适中。

### 整体布局要求

采用经典的侧边栏+主内容区布局:
- 左侧:固定宽度的全局导航栏(可收起)
- 右侧:主内容区,包含功能快捷入口、项目列表、投标日历
- 顶部右上角:用户登录状态

布局结构如下:
```
┌────────────────────────────────────────────────────────┐
│ [全局导航]  │                                    [👤用户]│
│   侧边栏    │     [功能快捷入口区]                        │
│            │    [项目列表区域]                  [投标日历]│
└────────────────────────────────────────────────────────┘
```

### 设计风格要求

- **整体风格**: 现代化B端产品风格,类似阿里云、腾讯云的管理后台
- **色彩方案**: 
  - 主色:#1890FF(蓝色,用于按钮、链接、高亮)
  - 辅助色:#52C41A(绿色,成功状态)、#FAAD14(橙色,警告)、#F5222D(红色,重要/截止)
  - 背景色:#F0F2F5(浅灰背景)、#FFFFFF(卡片白底)
  - 文字色:#262626(主文字)、#8C8C8C(次要文字)
- **间距**: 使用8px栅格系统(8、16、24、32、40px)
- **圆角**: 统一使用4px圆角
- **阴影**: 卡片使用 box-shadow: 0 2px 8px rgba(0,0,0,0.08)

### 技术要求

- 使用Vue + Ant Design
- 响应式设计,适配1920px、1440px、1366px屏幕
- 所有交互需要hover、active态反馈
- 数据加载时显示骨架屏(Skeleton)
- 使用虚拟滚动优化长列表性能

## 模块1: 全局导航栏

创建一个垂直侧边栏导航,包含以下功能:

**导航菜单项**(按顺序):
1. 首页 - 图标:🏠 (当前选中状态)
2. AI编标 - 图标:✨
3. 标书检查 - 图标:✅
4. 标讯 - 图标:📢
5. 项目管理 - 图标:📊
6. 电子证照库 - 图标:🏛️
7. 我的素材 - 图标:📚
8. 企业信息 - 图标:🏢

**交互要求**:
- 宽度:展开时240px,收起时64px
- 底部有"收起/展开"按钮
- 当前选中项有明显视觉反馈(背景色变化+左侧蓝色竖条)
- hover时显示浅色背景
- 收起后显示图标,展开后显示"图标+文字"

**样式细节**:
- 背景色:#001529(深蓝黑)

```

- 底部显示分页器或"加载更多"按钮
- 使用骨架屏实现加载态

**空状态设计**:
当没有项目时显示:

## 🎨 完整页面效果描述

整合以上所有模块后,页面应该呈现这样的效果:

1. **布局平衡**:左侧导航240px,右侧主内容区占据剩余空间,投标日历固定宽度400px在最右侧

2. **视觉层次**:功能快捷入口在顶部,项目列表在左侧占据主要空间,投标日历在右侧提供时间维度的信息

3. **色彩协调**:以白色、浅灰、蓝色为主色调,红色、橙色用于强调紧急信息,绿色用于成功状态

4. **交互流畅**:所有可点击元素有明确的hover反馈,过渡动画使用0.3s ease-in-out

5. **信息密度**:在保持可读性的前提下,一屏内展示尽可能多的有效信息

6. **响应式适配**:
   - 1920px:所有元素完整显示
   - 1440px:日历宽度缩小至350px
   - 1366px:日历移至项目列表下方

## 🔧 技术实现建议

**推荐技术栈**:
```
- React 18+
- Tailwind CSS 3+ 或 Ant Design 5+
- React Query(数据获取)
- date-fns(日期处理)
- react-window(虚拟滚动)
```

**组件拆分建议**:
```
HomePage/
  ├── GlobalNavigation/        # 全局导航
  ├── QuickActions/            # 功能快捷入口
  ├── UserProfile/             # 用户登录状态
  ├── ProjectList/             # 项目列表
  │   ├── ProjectFilters/      # 筛选栏
  │   ├── ProjectCard/         # 项目卡片
```

## 📝 使用方法

1. **完整生成**:将"核心提示词"部分完整复制给AI,生成完整页面

2. **分模块生成**:
   - 先生成基础布局
   - 逐个添加模块1-7
   - 最后整合优化

3. **迭代优化**:
   - 生成初版后,根据"验收检查清单"逐项检查
   - 针对不符合项,用具体的模块提示词重新生成
   - 调整细节样式和交互

4. **适配需求**:
   - 如需要深色模式,补充:"添加深色模式支持,背景#1F1F1F,卡片#2D2D2D"
   - 如需要移动端,补充:"添加移动端适配,768px以下使用单列布局"
