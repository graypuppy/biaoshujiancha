💡 给你的几个核心建议
1. 要学会"描述需求"而不是"写代码"
你用 AI 写代码的效率，取决于你 描述需求的能力。这正是你的优势——7年设计经验让你知道什么是好的交互，用这个能力去驾驭 AI。

示例 Prompt：

请帮我用 Vue 3 + Element Plus 实现检查结果列表页面。
需求：
1. 顶部有状态卡片，根据检查结果显示不同颜色和文案
2. 下面是表格，列有：序号、要求描述、检查结果、相关、结果
3. 检查结果列要用不同颜色图标区分：绿色对勾=通过，红色叉=不通过
4. 表格支持展开子级
5. 有筛选功能，可以筛选"仅不通过"
数据结构如下：
{检查点ID、名称、状态、原因、来源页码、是否否决项}
2. 把 PRD 当作喂给 AI 的"上下文"
你的 PRD 写得很详细，这就是你最好的资产。直接把 PRD 内容贴给 AI，让它帮你实现。

3. 不要追求"标准代码"，追求"能跑的代码"
先跑通，再优化。前端朋友可以帮你后续 Code Review，现在优先保证功能。

4. 遇到问题先找AI解决
报错信息直接贴给 AI，80%的问题它能帮你解决。解决不了再找前端朋友。
