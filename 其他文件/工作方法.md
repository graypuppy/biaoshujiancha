# 具体工作方法建议

### 3.1 需求拆分工作坊(1-2天)

**参与人员**: 产品经理、技术负责人、架构师、测试负责人

**工作流程**:

**Day 1 上午: 领域划分**
1. 使用白板/在线协作工具绘制功能地图
2. 识别天然的业务边界(资信标、技术标、经济标等)
3. 标注模块间的依赖关系
4. 投票决定优先级

**Day 1 下午: 阶段划分**
1. 基于技术依赖和业务价值,规划3个阶段
2. 每个阶段明确MVP范围
3. 识别技术风险点,调整阶段安排

**Day 2 上午: 详细拆分**
1. 针对阶段一,细分子模块
2. 评估每个子模块的工作量(故事点)
3. 调整拆分粒度,确保每个PRD可以在1-2周内完成

**Day 2 下午: 输出物确认**
1. 确定PRD清单与编写责任人
2. 确定评审流程
3. 确定交付时间表

### 3.2 单个PRD的编写流程(建议3-5天)

**Day 1: 需求澄清与结构搭建**
- 上午: 与利益相关方访谈,澄清需求
- 下午: 搭建PRD骨架,填充"概述"、"需求分析"章节

**Day 2-3: 功能设计**
- 使用AI辅助生成初稿(见下节)
- 人工审阅与精炼
- 绘制流程图、原型图

**Day 4: 非功能需求与测试设计**
- 技术评审,确定非功能需求
- 编写测试要点

**Day 5: 评审与修订**
- 内部评审
- 修订完善

### 3.3 如何高效使用AI生成PRD内容

**策略1: 分段生成,逐步精炼**
```
第一轮Prompt(生成骨架):
"我需要编写[资信标检查模块]的PRD,目标用户是[投标企业],
核心功能是[自动检查投标文件的资格符合性]。
请帮我生成PRD的章节目录和每个章节的核心内容提纲。"

第二轮Prompt(填充具体功能):
"基于上述目录,请详细设计[3.2.1 营业执照检查功能],
包含:用户故事、业务流程、业务规则、异常处理、验收标准。
业务背景是:[粘贴Charter中的相关内容]"

第三轮Prompt(优化与补充):
"请检查上述设计,补充以下方面:
1. 边界条件处理(如营业执照过期、图片模糊等)
2. 与招标文件解析模块的接口
3. 错误提示文案(要友好、可操作)"
